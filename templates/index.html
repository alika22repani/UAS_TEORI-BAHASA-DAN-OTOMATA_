Memvalidasi struktur tag HTML dengan Pushdown Automata

Setiap tag pembuka yang perlu ditutup dengan urutan yang tepat adalah persoalan bahasa bebas konteks klasik. Aplikasi ini menunjukkan bagaimana sebuah PDA membaca, men-push, dan men-pop tag HTML — langkah demi langkah, secara nyata.

Yang Bisa Dilakukan Aplikasi Ini

Satu mesin, empat lapis penjelasan

Bukan sekadar mengatakan "valid" atau "tidak valid" — aplikasi ini membuka seluruh proses internal PDA agar konsep teori otomata terasa konkret.

Validasi Struktur

Menentukan valid atau tidaknya struktur tag HTML secara instan.

Letak Kesalahan

Menunjukkan baris dan kolom persis tempat kesalahan tag ditemukan.

Simulasi Stack

Melihat isi stack berubah pada setiap operasi PUSH dan POP.

Jejak State

Menelusuri state (q_scan, q_accept, q_reject) yang dilalui mesin.

Cara Kerja

Dari kode HTML menjadi keputusan PDA

Empat tahap sederhana yang terjadi di balik layar setiap kali Anda menekan tombol validasi.

Baca penjelasan lengkap
01
Tokenisasi

Kode HTML dipecah menjadi token tag pembuka, penutup, dan tag mandiri.

02
Inisialisasi Stack

Stack dimulai dengan simbol dasar Z0 dan state q_scan.

03
Transisi PUSH / POP

Tag pembuka di-push, tag penutup memicu pop dan dicocokkan dengan puncak stack.

04
Keputusan Akhir

Diterima jika stack kembali ke Z0, ditolak jika ada tag yang tidak berpasangan.
